Переверстать сайт **www.muse.ooo** с Bootstrap 3 (тема Stack) на Tailwind CSS.

3. Деплой: GitHub → Vercel
4. Интеграцию с Bitrix будет делать программист (не в рамках этого проекта)

## Технические особенности

### CSS

- **Для разработки:** Tailwind CDN (`<script src="https://cdn.tailwindcss.com">`)
- **Для продакшена:** Скомпилированный `output.css`
- **Настройки:** Цвета, контейнер, типографика в `tailwind.config.js`

### Скрипты

- **Общие функции:** `js/scripts.js` (карусель, навигация, формы)
- **Tailwind Plus Elements:** CDN для мобильного меню

### Брейкпоинты (стандартные Tailwind)

| Класс | Ширина | Использование |
|-------|--------|---------------|
| `sm` | 640px | Мобильные (горизонтально) |
| `md` | 768px | Планшеты |
| `lg` | 1024px | Ноутбуки |
| `xl` | 1280px | Большие мониторы |

### Контейнер

- Максимальная ширина: **1170px**
- Padding: **15px** с каждой стороны
- Класс: `container`

## Деплой

### Быстрый способ

Запустить **`БЫСТРЫЙ_ДЕПЛОЙ.bat`** — он сделает всё автоматически.

### Команды

```bash
npm install          # Установка зависимостей
npm run build        # Сборка CSS (с отслеживанием)
npm run build:once   # Сборка CSS (один раз)
npm run preview      # Локальный просмотр
```

### Ссылки

- **Сайт на Vercel:** https://muse-liard-one.vercel.app/

## Подготовка к продакшену

Перед передачей программисту Битрикса:

1. **Запустить сборку:** `npm run build:once`
2. **Заменить CDN на output.css** в каждой странице
3. **Проверить работу** всех интерактивных элементов
4. **Передать документацию:** см. [docs/INTEGRATION_BITRIX.md](docs/INTEGRATION_BITRIX.md)

## Защита от индексации

Все страницы закрыты от индексации:
- `robots.txt` — запрещает индексацию
- `<meta name="robots" content="noindex, nofollow">` — в каждой странице

**Убрать защиту** только после полного переноса на Bitrix.
