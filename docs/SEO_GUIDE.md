# Руководство по управлению индексацией

Сейчас сайт закрыт от индексации поисковыми системами двумя способами:

1.  **Файл `robots.txt`**
    -   **Расположение:** `src/html/robots.txt`
    -   **Содержимое:** `Disallow: /`

2.  **Meta-тег `noindex, nofollow`**
    -   **Расположение:** В `<head>` всех HTML-страниц.
    -   **Код:** `<meta name="robots" content="noindex, nofollow">`

---

## Как убрать защиту от индексации

Эти шаги нужно выполнить **после** полного переноса сайта на production-сервер (например, после интеграции с Bitrix).

### Шаг 1: Изменить `robots.txt`

Откройте файл `src/html/robots.txt` и замените его содержимое, чтобы разрешить индексацию:

```
User-agent: *
Allow: /

Sitemap: https://muse.ooo/sitemap.xml
```

> **Примечание:** Путь к `sitemap.xml` следует указать после его генерации на стороне Bitrix.

### Шаг 2: Удалить meta-теги `noindex`

Необходимо удалить или закомментировать следующую строку из **всех** HTML-файлов в проекте:

```html
<!-- Закрыто от индексации во время разработки -->
<meta name="robots" content="noindex, nofollow">
```

Это можно сделать массовым поиском и заменой в вашем редакторе кода.

### Шаг 3: Развернуть изменения

После внесения изменений выполните деплой, чтобы обновить сайт на Vercel или production-сервере.

1.  Добавьте изменения в Git: `git add .`
2.  Создайте коммит: `git commit -m "feat: Allow site indexing for production"`
3.  Отправьте на сервер: `git push`

---

## Проверка

-   **До изменений:** Сайт закрыт для поисковых роботов.
-   **После изменений:** Сайт будет доступен для индексации и со временем появится в результатах поиска.
