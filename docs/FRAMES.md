## 1. Назначение

`frames.js` — единый модуль каталога багетных рам, подключаемый **до** `calc.js` / `modular-calc.js` / `reproduction-search.js`. Содержит:

- Массив из **39 реальных багетов** + 1 запись «Без багета» (`DEFAULT_FRAMES`)
- DOM-рендер текстурных рам (Подход B — strip + corner текстуры)
- Утилиты: расчёт цены, CSS-превью (fallback), хэш-таблица, форматирование чисел

Для управления каталогом (добавление/удаление рам, изменение цен, требования к изображениям) см. [§11](#11-руководство-по-управлению-каталогом-багетов).

## 2. Подключение (порядок скриптов)

```html
<script src="js/nav.js" defer></script>
<script src="js/uploader.js" defer></script>
<script src="js/prices.js" defer></script>
<script src="js/frames.js" defer></script>        <!-- ← ДО calc.js -->
<script src="js/calc.js?v=3" defer></script>
```

`calc.js` при инициализации читает `window.MUSE_FRAMES.DEFAULT_FRAMES`. Если `frames.js` не загружен — ошибка.

## 5. Экспортируемые функции (`window.MUSE_FRAMES`)

| Экспорт | Сигнатура | Описание |
|---------|-----------|----------|
| `DEFAULT_FRAMES` | `Array` | Массив 39 реальных багетов + 1 NONE |
| `buildFrameMap(frames)` | `Array → Object` | Хэш-таблица `{ id → frame }` для быстрого поиска |
| `getFramePrice(frame, w, h)` | `→ number` | Цена рамы: `Math.round(0.18 × (w + h) × pricePerM)` |
| `getFramePreviewCSS(frame)` | `→ Object` | CSS-стили для fallback-превью: `{ border, boxShadow }` |
| `fmtPrice(n)` | `→ string` | Число → строка с пробелами (`12450` → `"12 450"`) |
| `getStrips(frame, cb)` | `async` | Загружает strip-текстуру, создаёт 4 поворота (toBlob + ObjectURL), вызывает `cb(strips)` |
| `renderFrameDOM(container, frame, imgEl)` | `→ void` | Собирает DOM-раму из 8 div (4 стороны + 4 угла) вокруг `imgEl` |
| `revokeAll()` | `→ void` | Освобождает все blob ObjectURL (вызывать на `beforeunload`) |

### Формула цены

```
Цена = Math.round(0.18 × (ширина_см + высота_см) × frame.pricePerM)
```

Подробнее: [§11.2](#112-как-изменить-цены)

### CSS-превью (fallback `getFramePreviewCSS`)

Используется как fallback если текстуры не загрузились, и в `reproduction-search.js`.

1. `frame.id === 'NONE'` → `border: none`, тень без рамки
2. Ширина clamped: `Math.max(4, Math.min(frame.width, 20))` px
3. Остальные → `border: Npx solid <frame.color>`

## 6. UX-процесс выбора рамы (в calc.js)

1. Пользователь видит секцию `#frame-section` с кнопкой «Выбрать багет»
2. Клик → открывается `<dialog id="frame-modal">`
3. В модалке — 3 вкладки: **FLAT** (`#frames-grid-flat`), **STUDIO** (`#frames-grid-studio`) и **CLASSIC** (`#frames-grid-classic`)
4. Карточки загружаются лениво (IntersectionObserver) — strip-текстуры и повороты создаются при прокрутке
5. Каждая рама — мини-текстурный рендер + название + цена
6. Пользователь выбирает раму → модалка закрывается
7. На превью фото — DOM-рама из текстур (`renderFrameDOM`)
8. Цена рамы добавляется к итоговой стоимости
9. Кнопка «Без багета» в футере модалки сбрасывает выбор

### DOM-элементы (генерируются `buildFrameSections()` в calc.js)

| ID | Роль |
|----|------|
| `frame-section` | Родительская секция |
| `frame-modal` | `<dialog>` каталога |
| `frames-grid-studio` | Grid контейнер STUDIO |
| `frames-grid-classic` | Grid контейнер CLASSIC |

## 9. Рендеринг настоящих багетных текстур (Подход B — DOM-композиция)

> **Дата решения:** июль 2025
> **Статус:** ✅ Интегрировано в калькулятор
> **Тестовая страница:** `src/html/baget-test.html`

### 9.1. Суть решения

Вместо CSS-бордеров рама вокруг фотографии собирается в **браузере** из настоящих WebP-текстур багета:

- **Полоска** (`1.webp`) — вертикальный фрагмент профиля, повторяемый вдоль стороны
- **Уголок** (`2.webp`) — угловое соединение (под 45°)

Сервер только хранит картинки. Никакого серверного рендеринга не требуется.

### 9.2. Сравнение подходов (протестированы все три)

| Подход | Описание | Вердикт |
|--------|----------|---------|
| **A — border-image** | CSS `border-image` + `border-image-slice` | Работает, но сложно управлять углами |
| **B — DOM-композиция** ✅ | 8 `<div>` (4 стороны + 4 угла) с `background` и `transform` | **Выбран** — прост, нативен, легко интегрируется |
| **C — Canvas** | `<canvas>` + `drawImage()` + `createPattern()` | Работает, но избыточная сложность |

Все три визуально идентичны. Выбран B за простоту поддержки.

### 9.3. Архитектура DOM-рендера (Подход B)

```
┌──────────────────────────────────────┐
│            frame-wrapper             │  position: relative
│  ┌─────┬────────────────────┬─────┐  │
│  │ TL  │       TOP          │ TR  │  │  corner / strip-x
│  ├─────┼────────────────────┼─────┤  │
│  │     │                    │     │  │
│  │ L   │    <img> photo     │  R  │  │  strip-y / content
│  │     │                    │     │  │
│  ├─────┼────────────────────┼─────┤  │
│  │ BL  │      BOTTOM        │ BR  │  │  corner / strip-x
│  └─────┴────────────────────┴─────┘  │
└──────────────────────────────────────┘
```

**8 элементов:**
- 4 стороны (`<div>`) — `background: url(strip) repeat-x/y`, размер и трансформация
- 4 угла (`<div>` с `<img>`) — масштабирование через `transform: scale()`

### 9.4. Геометрия полоски (1.jpeg)

Исходная картинка — **левая** вертикальная сторона рамы:
- Правый край = **внутренний** (к картине)
- Левый край = **внешний**

| Сторона | Трансформация | Повтор |
|---------|---------------|--------|
| Left | as-is | `repeat-y` |
| Right | `scaleX(-1)` | `repeat-y` |
| Top | поворот **+90° CW** | `repeat-x` |
| Bottom | поворот **−90° CCW** | `repeat-x` |

> ⚠️ Направление поворота критично: при +90° CW внутренний край (правый в оригинале) оказывается внизу — к картине. При -90° CCW — вверху, тоже к картине.

### 9.5. Геометрия уголка (2.jpeg)

Исходный уголок — **верхний левый** (top-left):

| Угол | Трансформация |
|------|---------------|
| TL (top-left) | as-is |
| TR (top-right) | `scaleX(-1)` |
| BL (bottom-left) | `scaleY(-1)` |
| BR (bottom-right) | `scale(-1, -1)` |

### 9.6. Ширина рамы в пикселях

Реальная ширина профиля каждого багета хранится в поле `widthMm`. Для экранного отображения:

```
frameW_px = widthMm × 0.8
```

Коэффициент 0.8 подобран эмпирически для баланса между реалистичностью и удобством превью.

## 10. Оптимизация производительности

### 10.1. Проблема

При открытии модального окна багетов одновременно:
- Загружаются 39 strip-текстур
- Для каждой создаются 4 canvas-поворота → ~156 операций `toDataURL`
- Все dataURL (~100KB+ каждый) хранились как строки в JS-heap

### 10.2. Реализованные оптимизации

| Стадия | Что сделано | Файл |
|--------|-------------|------|
| **1A** | `loading="lazy"` + `decoding="async"` на corner `<img>` | `frames.js`, `calc.js` |
| **1B** | `content-visibility: auto` на скрытых grid-секциях (STUDIO, CLASSIC) | `input.css` |
| **2C** | `canvas.toBlob()` + `URL.createObjectURL()` вместо `toDataURL()` | `frames.js` |
| **2D** | `IntersectionObserver` — текстуры загружаются при прокрутке к карточке | `calc.js` |
| **2E** | `revokeAll()` — освобождение blob-памяти на `beforeunload` | `frames.js`, `calc.js` |
| **3F** | JPEG/PNG → WebP (100×100, quality 80) — **305 KB → 109 KB (−64%)** | `scripts/optimize-bagets.js` |

### 10.3. Архитектура после оптимизации

```
Открытие модального окна
  → DOM карточек строится сразу (fallback цвет фона)
  → IntersectionObserver наблюдает за .frame-option
  → Карточка входит во viewport (rootMargin: 200px)
    → _loadStrips() → getStrips(frame, cb)
      → Загрузка 1.webp (strip, ~0.2–3.6 KB)
      → 4× _makeRotated() → canvas.toBlob() → URL.createObjectURL()
      → Результат кешируется в _stripCache
      → DOM-рендер: 4 div + 4 corner img (2.webp, lazy)
  → beforeunload → revokeAll() → освобождение blob-памяти
```

### 10.4. Оптимизация изображений (Stage 3F)

Исходные текстуры (100×100px JPEG/PNG) конвертированы в WebP quality 80.

| Метрика | Значение |
|---------|----------|
| Файлов обработано | 78 (39 strip + 39 corner) |
| Формат до | JPEG (72 шт.) + PNG (6 шт.) |
| Формат после | WebP (78 шт.) |
| Размер до | 305 KB |
| Размер после | 109 KB |
| Экономия | 196 KB (−64%) |
| Лучший результат | PNG рамы (39-22-10, 39-22-6, 47-26-1): −87–97% |

**Скрипт:** `npm run optimize:bagets` (`scripts/optimize-bagets.js`)
**Backup оригиналов:** `draft/Багет/`

> **Правило:** правки калькулятора — сначала в `calc.html` (источник истины), потом распространять на продуктовые страницы.

## 11. Руководство по управлению каталогом багетов

### 11.1. Требования к изображениям для рендеринга

Каждый багет требует **2 текстуры**, расположенных в `src/html/img/bagets/{id}/`:

| Файл | Назначение | Как используется |
|------|------------|------------------|
| `1.webp` | **Полоска** (strip) — вертикальный фрагмент профиля | Повторяется `repeat-y` (Left), зеркалится (Right), поворачивается ±90° (Top, Bottom) |
| `2.webp` | **Уголок** (corner) — угловое соединение 45° | Отображается 4 раза: as-is (TL), flip-X (TR), flip-Y (BL), flip-XY (BR) |

**Требования к файлам:**

| Параметр | Значение |
|----------|----------|
| Формат | **WebP** (конвертация через `npm run optimize:bagets`) |
| Размер | **100 × 100 px** |
| Качество | 80 (параметр скрипта `optimize-bagets.js`) |
| Ориентация полоски | Вертикальная — левая сторона рамы. Правый край = внутренний (к картине), левый = внешний |
| Ориентация уголка | Верхний левый угол (top-left) |
| Фон | Без прозрачности (непрозрачный фон) |

**Если нет WebP-исходников:**

1. Положите исходные JPEG или PNG файлы как `1.jpeg`/`1.png` и `2.jpeg`/`2.png` в `src/html/img/bagets/{id}/`
2. Запустите `npm run optimize:bagets` — скрипт автоматически сконвертирует в WebP
3. Если WebP оказался больше оригинала, скрипт оставит оригинальный формат (но такое случается редко)

**Fallback:** если текстура не загрузилась, рендер использует CSS-бордер цвета из поля `color`.

### 11.2. Как изменить цены

Цены определяются полем `pricePerM` (руб./погонный метр) в массиве `DEFAULT_FRAMES` файла `src/html/js/frames.js`.

**Формула расчёта цены рамы:**

```
Цена = Math.round(0.18 × (ширина_см + высота_см) × pricePerM)
```

Где `0.18` — коэффициент пересчёта (периметр × наценка), `ширина_см` и `высота_см` — размеры, введённые пользователем.

**Пример:** рама `33-24-1` с `pricePerM: 115` при размере 60×40 см:

```
Math.round(0.18 × (60 + 40) × 115) = Math.round(2070) = 2 070 ₽
```

**Порядок изменения цены:**

1. Откройте `src/html/js/frames.js`
2. Найдите нужный багет в массиве `DEFAULT_FRAMES` (строки ~20–65)
3. Измените значение `pricePerM`
4. Сохраните, деплойте — пересборка CSS не нужна, изменения чисто JS

**Массовое изменение** — если нужно поменять цену всей группе, ищите по `pricePerM: <старое_значение>` и заменяйте. Например, все плоские (FLAT) сейчас `pricePerM: 70`.

### 11.3. Как добавить новый багет

**Шаг 1 — Подготовить текстуры**

1. Сфотографируйте / отсканируйте багет. Нужны 2 картинки:
   - Полоска профиля (вертикальный срез, левая сторона рамы)
   - Уголок (стык 45°, верхний левый)
2. Обрежьте оба до **100 × 100 px**
3. Сохраните как `1.jpeg` (или `.png`) и `2.jpeg` (или `.png`)
4. Создайте папку `src/html/img/bagets/{новый-id}/` и положите туда файлы
5. Запустите `npm run optimize:bagets` — будут сконвертированы в WebP

**Шаг 2 — Добавить запись в каталог**

Откройте `src/html/js/frames.js` и добавьте объект в массив `DEFAULT_FRAMES` в соответствующую группу:

```javascript
{
  id: '33-24-6',           // Уникальный ID = имя папки с текстурами
  name: '33-24-6',         // Отображаемое имя в UI (обычно = id)
  cat: 'CLASSIC',          // Категория: FLAT | STUDIO | CLASSIC
  group: 'классический',   // Группа: плоский | студийный | классический
  widthMm: 33,             // Ширина профиля в мм (из параметров поставщика)
  width: 26,               // Экранная ширина в px ≈ widthMm × 0.8
  pricePerM: 115,          // Цена за погонный метр (руб.)
  stripUrl: 'img/bagets/33-24-6/1.webp',
  cornerUrl: 'img/bagets/33-24-6/2.webp',
  imageUrl: 'img/bagets/33-24-6/2.webp',  // Превью (обычно = cornerUrl)
  color: '#8B7355',        // Fallback-цвет CSS-бордера
  available: true           // true = показывать в каталоге
}
```

**Справочник полей:**

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `string` | Уникальный идентификатор. Должен совпадать с именем папки в `img/bagets/` |
| `name` | `string` | Название в UI (модальное окно каталога) |
| `cat` | `string` | Категория: `FLAT`, `STUDIO` или `CLASSIC` — определяет вкладку в модальном окне |
| `group` | `string` | Текстовая группа: `плоский`, `студийный`, `классический` |
| `widthMm` | `number` | Реальная ширина профиля в мм |
| `width` | `number` | Экранная ширина рамы в px (формула: `Math.round(widthMm × 0.8)`) |
| `pricePerM` | `number` | Цена за погонный метр (руб.). `null` для «Без багета» |
| `stripUrl` | `string` | Путь к текстуре полоски (`1.webp`) относительно HTML |
| `cornerUrl` | `string` | Путь к текстуре уголка (`2.webp`) относительно HTML |
| `imageUrl` | `string` | Превью-картинка (обычно = `cornerUrl`) |
| `color` | `string` | Hex-цвет для CSS-fallback если текстура не загрузилась |
| `available` | `boolean` | `true` — показывать, `false` — скрыть из каталога |

**Шаг 3 — Проверить**

1. Откройте `calc.html` (или любую продуктовую страницу) в браузере
2. Нажмите «Выбрать багет» → убедитесь, что новый багет виден в нужной вкладке
3. Проверьте, что текстура отображается корректно (полоска по сторонам, уголки в углах)

**Соглашение по ID:** формат `ШиринаМм-серия-номер` (например `33-24-6`, `25-981-12`).

### 11.4. Как удалить / отключить багет

#### Вариант А — Мягкое отключение (рекомендуется)

Поставьте `available: false` — багет скроется из UI, но данные сохранятся:

```javascript
{ id: '33-24-1', ..., available: false }
```

- Рама не появляется в модальном окне каталога
- Если пользователь ранее выбрал эту раму (из сохранённого URL), код обработает это как «Без багета»
- Легко вернуть обратно — поменять `false` на `true`

#### Вариант Б — Полное удаление

1. Удалите объект из массива `DEFAULT_FRAMES` в `frames.js`
2. Удалите папку `src/html/img/bagets/{id}/`
3. Деплойте

**Рекомендация:** используйте Вариант А для временного снятия (закончился на складе, снят с производства). Вариант Б — только если багет точно больше никогда не вернётся.
