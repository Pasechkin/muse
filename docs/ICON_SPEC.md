## 1. Проблема

Маркетинговые иконки (секция «Преимущества» и аналоги) импортированы из разных источников и **не образуют единый стиль.**

### Текущие проблемы

| Проблема | Пример |
|----------|--------|
| **35+ различных viewBox** | `0 0 19.3 19.3`, `0 0 612 612`, `0 0 2119 1889`, `314 155 1743 1743` |
| **Смешение fill и stroke** | icon-1…45 = fill (clipart); icon-46…57 = stroke (Heroicons) |
| **Непрямоугольные пропорции** | icon-13: 2119×1889, icon-64: 1024×1536, icon-45: 338×432 |
| **Нестандартный offset** | icon «рука-кисть» viewBox `314 155 1743 1743`, icon-27 viewBox `-50 -36 4554 4550` |
| **Разная плотность деталей** | Мелкие clipart vs крупные outline — при 1.875rem выглядят хаотично |
| **Дублирование каталогов** | `optimized/` (81), `production/` (75), `unified/` (19) — синхронизация не гарантирована |

## 2. Целевая спецификация

### 2.1. Grid и viewBox

```
viewBox="0 0 30 30"
```

- **Всегда квадратный:** 30×30 px grid
- **Offset всегда 0 0** — никаких `314 155 …` или отрицательных значений
- Визуальное содержимое в пределах safe area **2px padding** (контент в зоне 2–28)

### 2.2. Стиль: Line-art outline monostroke

```
stroke="currentColor"
stroke-width="2"
stroke-linecap="round"
stroke-linejoin="round"
fill="none"
```

| Параметр | Значение | Обоснование |
|----------|----------|-------------|
| Тип | **Stroke-based** (не fill) | Единообразие с UI-иконками, управление цветом через `currentColor` |
| stroke-width | **2px** | Оптимально для рендеринга при 1.875rem (30px); не теряется на мелких, не жирнит на крупных |
| linecap | **round** | Мягкий стиль, гармонирует с rounded-UI дизайна Muse |
| linejoin | **round** | Совместимо с linecap round |
| fill | **none** | Только обводка, без заливки (monostroke) |

### 2.3. Цвет (управление через CSS)

Иконки **не должны** содержать жёстких цветов (`fill="#D97706"`). Всё через `currentColor`:

```html
<!-- HTML -->
<svg class="icon-advantage" viewBox="0 0 30 30" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round">
  <path d="..."/>
</svg>
```

```css
/* CSS (уже есть в input.css) */
.icon-advantage {
  width: 1.875rem;   /* 30px */
  height: 1.875rem;
  color: var(--color-ah-950);
}
```

### 2.4. Размер отображения

| Контекст | CSS-класс | Размер | Примечание |
|----------|-----------|--------|------------|
| Преимущества | `.icon-advantage` | 1.875rem (30px) | Внутри `.advantage-card__icon` (3.625rem circle, bg-ah-50) |
| UI Controls | `.ui-control--sm/md/lg/xl` | 16–32px | Отдельная система, не затрагивается |
| Соцсети | `w-5 h-5` / `w-6 h-6` | 20–24px | Brand-иконки, не редизайн |

## 4. Задание на перерисовку — пошаговый чеклист

### 4.1. Параметры каждого готового файла

Каждый перерисованный SVG **ОБЯЗАН** соответствовать этому шаблону:

```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 30 30"
     fill="none" stroke="currentColor" stroke-width="2"
     stroke-linecap="round" stroke-linejoin="round">
  <!-- paths here -->
</svg>
```

### 4.2. Правила отрисовки

1. **Стиль — monostroke line-art.** Только обводка, без заливки. Один вес линии (stroke-width 2) на всю иконку.
2. **Safe area.** Визуальный контент вписан в зону 2–28 по обеим осям (padding 2px от края viewBox).
3. **Без жёстких цветов.** Никаких `fill="#D97706"` или `stroke="#000"`. Только `currentColor`.
4. **Без transform.** Никаких `transform="translate(…)"`, `scale(…)`, `matrix(…)`. Все координаты — абсолютные в системе 0–30.
5. **Минимум path'ов.** Упрощать: где можно заменить сложный path на `<circle>`, `<rect>`, `<line>`, `<polyline>` — использовать примитивы.
6. **Без `<g>`, `<defs>`, `<clipPath>`, `<mask>`.** Плоская структура: только `<path>`, `<circle>`, `<rect>`, `<line>`, `<polyline>`.
7. **Неквадратные оригиналы** (icon-12, 13, 24, 45, 59, 60, 64, 78, 80): перекомпоновать сюжет в квадрат 30×30, а не просто растянуть.
8. **Узнаваемость.** Перерисованная иконка должна передавать тот же сюжет, что и оригинал (см. столбец «Сюжет» в таблицах §3.1 и §3.2).

### 4.3. Пример: щит с галочкой (icon-6)

**Было** (fill, viewBox 214.3×214.3, ~800 байт path data):
```xml
<svg viewBox="0 0 214.3 214.3"><path d="M196.9 55.2c-.1-5.8-.2-11.2 …" fill="#000"/></svg>
```

**Стало** (monostroke, viewBox 30×30, ~120 байт):
```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 30 30"
     fill="none" stroke="currentColor" stroke-width="2"
     stroke-linecap="round" stroke-linejoin="round">
  <path d="M15 3 L5 8 V15 C5 21 9 26 15 28 C21 26 25 21 25 15 V8 Z"/>
  <polyline points="11 15 14 18 19 11"/>
</svg>
```

### 4.4. Подгонка stroke-иконок (Группа B, icon-46–56)

Механическая операция — масштабирование координат:

```
Новая координата = Старая координата × (30 / 24) = × 1.25
```

1. Умножить все числовые значения в `d="…"` на 1.25
2. Сменить `viewBox="0 0 24 24"` → `viewBox="0 0 30 30"`
3. Сменить `stroke-width="1.75"` → `stroke-width="2"`
4. Проверить, что контент остался в safe area 2–28

### 4.5. Оптимизация

После отрисовки прогнать каждый файл через SVGO:

```bash
npx svgo --config svgo.config.js src/icons/optimized/icon-N.svg
```

Конфиг `svgo.config.js` уже настроен в проекте.

### 4.6. Валидация (приёмка каждой иконки)

- [ ] `viewBox="0 0 30 30"` — ровно эти значения
- [ ] `fill="none"` на корневом `<svg>`
- [ ] `stroke="currentColor"` на корневом `<svg>`
- [ ] `stroke-width="2"` на корневом `<svg>`
- [ ] `stroke-linecap="round"` и `stroke-linejoin="round"`
- [ ] Нет вложенных `<g>`, `<defs>`, `<clipPath>`, `<mask>`
- [ ] Нет атрибутов `transform`
- [ ] Нет жёстких цветов (`fill="#…"`, `stroke="#…"`)
- [ ] Контент в зоне 2–28 (safe area)
- [ ] Визуально узнаваем оригинальный сюжет при отображении 30×30 px в круге bg-ah-50
- [ ] Файл прошёл SVGO без ошибок

## 8. Связь с CSS

Текущие CSS-классы (из `input.css`) **уже готовы** к приёму новых иконок:

```css
/* Контейнер */
.advantage-card__icon {
  width: 3.625rem;
  height: 3.625rem;
  border-radius: 9999px;
  background: var(--color-ah-50);
  display: flex;
  align-items: center;
  justify-content: center;
}

/* Иконка */
.icon-advantage,
.advantage-card__icon svg {
  width: 1.875rem;
  height: 1.875rem;
  color: var(--color-ah-950);
  fill: var(--color-ah-950);
}
```

> **Внимание:** после перехода на stroke-based иконки нужно обновить CSS:
> - Убрать `fill: var(--color-ah-950)` из `.icon-advantage`
> - Добавить `stroke: currentColor; fill: none;` (или полагаться на атрибуты SVG)
