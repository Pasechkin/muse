═══════════════════════════════════════════════════════════
  ВАЖНО: Два файла для разных целей
═══════════════════════════════════════════════════════════

У вас теперь ДВА файла:

1. index.html
   └─ Использует внешний CSS: <link rel="stylesheet" href="css/output.css">
   └─ Для ЛОКАЛЬНОЙ разработки (открытие в браузере)
   └─ Работает только если есть файл css/output.css

2. index_tailwind.html  
   └─ Использует инлайновый CSS (CSS встроен в HTML)
   └─ Для ПРОДАКШЕНА (Vercel, GitHub Pages)
   └─ Работает без внешних файлов

═══════════════════════════════════════════════════════════

📋 ЧТО ДЕЛАТЬ:

✅ Для локальной работы:
   - Открывайте и редактируйте: src/html/index.html
   - Убедитесь, что CSS собран: запустите npm run build:once && npm run copy-css

✅ Для загрузки на сервер (Vercel):
   - Сначала синхронизируйте: запустите СИНХРОНИЗИРОВАТЬ_ФАЙЛЫ.bat
   - Это скопирует index_tailwind.html → index.html (с инлайновым CSS)
   - Затем загрузите на GitHub: запустите БЫСТРЫЙ_ДЕПЛОЙ.bat

═══════════════════════════════════════════════════════════

🔄 РАБОЧИЙ ПРОЦЕСС:

Локально:
1. Редактируете index.html
2. Открываете в браузере и проверяете
3. Если нужно - собираете CSS: npm run build:once && npm run copy-css

Перед загрузкой на сервер:
1. Копируете изменения из index.html в index_tailwind.html
   (или редактируете index_tailwind.html напрямую)
2. Запускаете СИНХРОНИЗИРОВАТЬ_ФАЙЛЫ.bat (для продакшена)
3. Запускаете БЫСТРЫЙ_ДЕПЛОЙ.bat (отправка на GitHub/Vercel)

═══════════════════════════════════════════════════════════

💡 РЕКОМЕНДАЦИЯ:

Для простоты можете работать ТОЛЬКО с index.html локально,
а перед каждым деплоем:
1. Скопировать index.html → index_tailwind.html
2. В index_tailwind.html заменить внешний CSS на инлайновый
   (можно создать скрипт для этого)

Или работайте сразу с index_tailwind.html и используйте
локальный сервер (npm run preview) для просмотра.
